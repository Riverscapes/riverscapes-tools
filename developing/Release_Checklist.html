Release Checklist

We’re moving to monthly releases.

  1. resolve any branch -> dev pull requests
  2. Increment version numbers if necessary
  3. Create a pull request to merge dev –> master
  4. Run every tool once through cybercastor staging on dev
  5. Quick code review
  6. Merge the PR dev –> master
